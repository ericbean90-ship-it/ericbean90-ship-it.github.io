## 열흘의 연습

지난 10일 동안 Cursor를 붙잡고 개발 연습을 했습니다. 시작은 단순했어요. AI가 코드를 써주는 시대에 그 흐름을 몸으로 익혀보고 싶었습니다. 이른바 **바이브 코딩(Vibe Coding)** — 완벽한 이해보다 흐름을 타며 만들어 나가는 방식이요.

처음에는 AI가 내놓는 코드를 하나하나 읽었습니다. 이게 맞는지, 더 좋은 방법은 없는지 살피면서요. 그런데 며칠이 지나자 이상한 걸 느꼈습니다. 코드를 꼼꼼히 볼수록 속도가 느려지고, 막연히 흐름을 믿고 나아갈 때 오히려 더 많은 것이 완성됐습니다.

## 코드를 볼 이유가 적다

바이브 코딩의 핵심을 이렇게 정리하게 됐습니다.

> AI가 쓴 코드를 내가 이해할 필요는 없다. 내가 원하는 것이 작동하는지가 중요하다.

물론 틀린 말은 아닙니다. 프로덕션 코드라면 당연히 이해와 검토가 필요하죠. 하지만 **아이디어를 빠르게 검증하는 단계**에서는 달랐습니다. 코드 한 줄 한 줄을 추적하는 것보다, 결과물이 내 의도대로 움직이는지를 보는 게 훨씬 효율적이었습니다.

에디터 화면을 가득 채운 코드보다, AI와 나누는 대화가 더 중요해졌습니다. Cursor의 패널 속 코드 에디터가 점점 배경으로 물러나는 느낌이었어요.

## 그래서 클로드 코드로

그 깨달음이 자연스럽게 다음 선택으로 이어졌습니다. 코드를 볼 이유가 적다면, 굳이 에디터 중심의 도구를 쓸 이유도 적다는 것이었습니다.

**클로드 코드(Claude Code)** 는 터미널에서 AI와 직접 대화하는 CLI 도구입니다. 에디터 없이, 파일 탐색기 없이, AI에게 맥락을 주고 원하는 것을 말하면 됩니다. Cursor가 에디터에 AI를 얹은 도구라면, 클로드 코드는 AI가 먼저이고 에디터는 필요할 때만 열면 되는 도구에 가깝습니다.

바이브 코딩으로 얻은 감각 — 코드보다 의도에 집중하는 것 — 과 더 잘 맞는다고 느꼈습니다.

## 연습의 의미

10일이 짧다면 짧은 시간이지만, 생각의 방향이 바뀌기엔 충분했습니다. 어떤 도구가 더 낫다는 이야기가 아닙니다. Cursor는 코드를 직접 다루고 싶은 사람에게 여전히 훌륭한 도구입니다.

다만 저는 지금 단계에서 코드보다 결과에 더 집중하고 싶습니다. 그 흐름에서 클로드 코드가 잘 맞는다고 느꼈고, 그래서 넘어가기로 했습니다.

연습은 계속됩니다.

## Ten Days of Practice

For the past ten days, I spent my time practicing development with Cursor. The motivation was simple — I wanted to feel firsthand what it means to build in an era where AI writes the code. This is what people call **Vibe Coding**: riding the flow rather than demanding perfect understanding at every step.

At first, I read through every line the AI produced. I checked whether it made sense, whether there was a better approach. But a few days in, something felt off. The more carefully I scrutinized the code, the slower everything got. When I loosened my grip and trusted the flow, more things actually got done.

## Less Reason to Look at the Code

That experience led me to a simple conclusion about vibe coding:

> I don't need to understand every line the AI writes. What matters is whether it does what I want.

This isn't true for all contexts — production code absolutely demands understanding and review. But at the stage of **quickly validating ideas**, it was different. Tracing line by line was far less useful than checking whether the output behaved as intended.

The conversation with the AI started to matter more than the code filling the editor. The code panel in Cursor gradually faded into the background.

## So, Claude Code

That realization led naturally to the next decision. If there's less reason to look at the code, there's also less reason to use an editor-centric tool.

**Claude Code** is a CLI tool that lets you talk directly to an AI from the terminal — no editor, no file explorer, just context and intent. Where Cursor puts AI on top of an editor, Claude Code puts AI first and lets you open an editor only when you need one.

That felt like a better fit for the instinct I developed through vibe coding: focus on intent, not code.

## What the Practice Meant

Ten days is a short time, but it was enough to shift how I think. This isn't a claim that one tool is better than the other. Cursor remains an excellent choice for anyone who wants to stay close to the code.

For where I am right now, I want to focus more on outcomes than on code. Claude Code felt like the right match for that, and so I made the move.

The practice continues.
