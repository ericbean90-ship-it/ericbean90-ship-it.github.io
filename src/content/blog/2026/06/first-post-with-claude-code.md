## 워크플로가 달라졌습니다

커서(Cursor)로 이 블로그를 만들었을 때는 글을 쓰는 방식도 따로 있었어요. 초고를 쓰고, ChatGPT로 다듬은 뒤, 그걸 다시 붙여넣어 올리는 흐름이었습니다. 크게 복잡한 과정은 아니었지만, 단계가 있었습니다. 도구가 여럿이었고, 오가는 흐름이 있었습니다.

클로드 코드로 넘어오면서 그 흐름이 달라졌습니다.

## 단계가 사라졌습니다

이 글은 클로드 코드에서 직접 썼습니다. 메모를 건네주고, 기존 글의 스타일을 참고하게 했고, 클로드가 초고를 완성했습니다. 따로 ChatGPT에서 다듬을 필요가 없었습니다. 클로드가 글쓰기도, 다듬기도, 파일 생성도, 커밋과 푸시도 한 번에 처리했습니다.

굳이 그럴 필요가 없어졌기 때문입니다.

커서 시절에는 도구마다 역할이 나뉘어 있었어요. 코드 작성은 커서, 글 다듬기는 ChatGPT, 배포는 따로. 클로드 코드는 그 경계가 없습니다. 대화 하나로 흐름 전체가 이어집니다.

## 작은 차이가 만드는 것

단계 하나가 줄었을 뿐인데, 글을 쓰는 일이 훨씬 가벼워졌습니다. 여러 도구를 오가는 마찰이 없으니, 메모에서 게시까지 막힘 없이 이어집니다. 더 많이 쓰게 될 것 같습니다.

클로드 코드로 하는 첫 글입니다. 앞으로 이 흐름으로 계속 써나갈 생각입니다.

## The Workflow Changed

When I built this blog with Cursor, I had a separate process for writing posts. Draft first, polish it with ChatGPT, then paste it back and publish. Not complicated, but there were steps — multiple tools, things moving back and forth.

Moving to Claude Code changed that.

## A Step Disappeared

This post was written directly in Claude Code. I handed over some notes, pointed it to the existing posts for style reference, and Claude put together the draft. No need to take it to ChatGPT for polishing. Claude handled the writing, the cleanup, the file creation, and the commit and push — all in one go.

Because there was simply no need for the extra step.

With Cursor, the tools each had their role: Cursor for code, ChatGPT for writing, a separate step for deployment. Claude Code blurs those lines. One conversation carries the whole flow.

## What a Small Difference Makes

Removing one step made writing feel noticeably lighter. Without the friction of switching between tools, the path from a rough note to a published post is uninterrupted. I think I'll end up writing more.

This is the first post written with Claude Code. I plan to keep going with this workflow.
